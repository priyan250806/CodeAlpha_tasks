## 📌 About This Project

This project was built as **Task 3** of the **CodeAlpha Cybersecurity Internship**.

**Objective:**
- Set up Suricata as a network-based IDS on Kali Linux
- Configure custom detection rules for common attack patterns
- Parse Suricata's `eve.json` log output using Python
- Visualize detected alerts on an interactive web dashboard

## ⚙️ Installation & Setup

### Step 1 — Run the automated setup script
```bash
chmod +x setup_suricata.sh
sudo bash setup_suricata.sh
```

This script automatically:
- Installs Suricata via `apt`
- Detects your active network interface
- Updates Suricata community rules
- Writes 10 custom detection rules
- Tests the configuration

### Step 2 — Start Suricata
```bash
# As a service (recommended)
sudo systemctl start suricata
sudo systemctl enable suricata

# Or run manually on your interface
sudo suricata -c /etc/suricata/suricata.yaml -i eth0
```

### Step 3 — Generate test traffic to trigger alerts
```bash
# Trigger port scan detection
nmap -sS 192.168.1.1

# Trigger SSH brute force detection
hydra -l root -P /usr/share/wordlists/rockyou.txt ssh://192.168.1.1

# Trigger SQL injection detection
curl "http://192.168.1.1/page?id=1' OR '1'='1"

# Trigger ICMP sweep
ping -c 20 192.168.1.1
```

### Step 4 — Parse logs and launch dashboard
```bash
# Parse Suricata eve.json → generates dashboard/data.json
python3 parser.py

# Open the dashboard in your browser
xdg-open dashboard/index.html
```

## 🔐 Custom Detection Rules

10 custom rules written in Suricata rule syntax (`custom.rules`):

| Rule ID | Threat Detected | Protocol | Threshold |
|---------|----------------|----------|-----------|
| 9000001 | Nmap SYN Port Scan | TCP | 20 SYN packets / 5 sec |
| 9000002 | SSH Brute Force | TCP/22 | 20 attempts / 5 sec |
| 9000003 | SQL Injection (OR 1=1) | HTTP | Any |
| 9000004 | SQL Injection (UNION SELECT) | HTTP | Any |
| 9000005 | XSS Attempt (`<script>`) | HTTP | Any |
| 9000006 | ICMP Ping Sweep | ICMP | 10 pings / 3 sec |
| 9000007 | General Port Scan | TCP | 30 SYN packets / 10 sec |
| 9000008 | Telnet Access (insecure) | TCP/23 | Any |
| 9000009 | FTP Login Attempt | TCP/21 | Any |
| 9000010 | RDP Brute Force | TCP/3389 | 10 attempts / 10 sec |

## 🔬 How It Works

```
Network Traffic (eth0/wlan0)
        │
        ▼
  Suricata IDS Engine
        │ matches rules
        ▼
  /var/log/suricata/eve.json
  (one JSON event per line)
        │
        ▼
  parser.py
  ├── Reads eve.json line by line
  ├── Filters event_type == "alert"
  ├── Aggregates: severity, category,
  │   src_ip, protocol, hourly counts
  └── Writes dashboard/data.json
        │
        ▼
  dashboard/index.html
  ├── Reads data.json via fetch()
  ├── Renders 8 Chart.js visualizations
  ├── Displays live alert table
  └── Auto-refreshes every 30 seconds
```

## 🔍 Useful Suricata Commands

```bash
# View live alerts in terminal (pretty JSON)
sudo tail -f /var/log/suricata/eve.json | python3 -m json.tool

# Check Suricata status
sudo systemctl status suricata

# View only alert events
sudo cat /var/log/suricata/eve.json | python3 -c \
  "import sys,json; [print(json.dumps(json.loads(l),indent=2)) \
  for l in sys.stdin if json.loads(l).get('event_type')=='alert']"

# Test your custom rules
sudo suricata -T -c /etc/suricata/suricata.yaml -v

# Update community rules
sudo suricata-update

# View fast.log (simple alert format)
sudo tail -f /var/log/suricata/fast.log
```
